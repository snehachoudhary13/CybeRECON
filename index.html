🔬 URL ANATOMY EXPLAINED

Protocol (HTTP/HTTPS): Determines how data is transmitted. HTTPS encrypts the connection using TLS. HTTP sends data as plaintext — any attacker on the same network can read it.

Domain: The human-readable address of a server. The domain reveals who owns the target and where it's hosted. You can use it to perform WHOIS lookups.

Port: If not shown, default ports apply — 80 for HTTP, 443 for HTTPS. Non-standard ports like :8080 or :3000 often indicate development servers exposed accidentally.

Path: Reveals the API structure. Paths like /api/v2/users tell an attacker which version of the API is running and what resource types exist.

Query Parameters: Key=value pairs after ?. Parameters like id=, user=, token= are prime targets for SQLi, IDOR, and parameter tampering.

Fragment (#): Processed client-side only, never sent to the server. Used in hash-based routing in Single Page Apps.

🔒 SSL/TLS CERTIFICATE EXPLAINED

Certificate Status (Valid/Invalid): A valid cert means the server's identity has been verified by a trusted Certificate Authority (CA). An invalid cert can mean it's expired, self-signed, or the domain doesn't match — all are red flags.

Issuer/CA: The Certificate Authority that signed the cert. Let's Encrypt is free and widely trusted. Unfamiliar issuers may indicate self-signed or rogue certificates used in phishing.

TLS Version: TLS 1.3 is the modern standard. TLS 1.2 is acceptable. TLS 1.0/1.1 are deprecated and vulnerable to POODLE and BEAST attacks.

Cipher Suite: The algorithm combination used for encryption. Strong: AES_256_GCM. Weak/dangerous: RC4, DES, 3DES — these can be brute-forced.

Expiry / Days Remaining: Expired certs break trust and warn users. Attackers can monitor expiry to time domain-hijack attacks or use expired certs as phishing indicators.

HSTS (Strict-Transport-Security): Forces browers to only use HTTPS for the domain. Without it, a downgrade attack can force the browser to use HTTP, exposing session tokens.

Security Grade (A+ / F): A combined score based on protocol version, key size, and cipher strength. Grade F means the server is critically insecure.

📡 HTTP SECURITY HEADERS EXPLAINED

Server: (e.g. nginx/1.18.0): Dangerous. Leaking the server software name and version lets attackers look up exact CVEs for that version on sites like NVD and Exploit-DB.

X-Powered-By: (e.g. Express, PHP): Dangerous. Reveals backend technology. Remove this header entirely in production. Attackers use it to fingerprint the tech stack.

Content-Security-Policy (CSP): Required. Tells the browser which scripts, styles, and resources are safe to load. Missing CSP allows Cross-Site Scripting (XSS) attacks to inject malicious code.

Strict-Transport-Security (HSTS): Required. Forces HTTPS. Without it, attackers can perform SSL stripping to intercept traffic on insecure networks.

X-Frame-Options: SAMEORIGIN: Prevents the page from being embedded in an iframe on another domain. Missing this allows Clickjacking — where a user thinks they're clicking one thing but actually interacts with a hidden iframe.

X-XSS-Protection: 0: Deprecated header, but setting it to 0 actually disables the browser's built-in (legacy) XSS filter. Rely on CSP instead of this header.

Security Score (e.g. 65/100): Calculated from presence/absence of key security headers. Aim for 90+. Each missing header reduces the score and widens the attack surface.

🗺️ IP GEOLOCATION EXPLAINED

Country / City: Geographic origin of the IP. A server claiming to be a legitimate business but hosted in a high-risk country is a red flag. OSINT investigators cross-reference this against corporate registration records.

ASN (Autonomous System Number): Identifies the network block owner. AS204778 can be looked up in ARIN/RIPE to find the organization controlling that IP range.

ISP / Organization: The internet provider. Hosting providers like OVH, Hetzner, or DigitalOcean are commonly abused for C2 (Command & Control) servers in malware campaigns.

Proxy / VPN / Tor flags: High risk. Attackers use Tor exit nodes and commercial VPN IPs to anonymize attacks. A flagged IP should be immediately investigated and possibly blocked.

Coordinates (Lat/Long): Not precise for end-user geolocation (usually city-level), but useful for identifying datacenter locations and correlating with known hosting infrastructure.

🕵️ WHOIS INTELLIGENCE EXPLAINED

Registrar: The company the domain was purchased through. Attackers prefer certain registrars known for lax abuse reporting, like Namecheap or Epik, when setting up phishing domains.

Created Date: Critical indicator. Domains less than 30 days old are a major phishing red flag. Legitimate companies have years-old domains; phishers spin up fresh ones.

Expiry Date / Domain Age: Calculated as years/months since registration. Very short registration periods (1 year) suggest a disposable phishing or malware domain.

Name Servers: DNS servers responsible for the domain. Unusual nameservers for a claimed brand (e.g., a "Google" domain using a random NS) indicates spoofing.

Privacy Protection (WhoisGuard): Hides registrant contact info. While legitimate businesses use it, it's also a common tactic for malicious actors to avoid attribution.

☠️ THREAT INTELLIGENCE EXPLAINED

IOC (Indicator of Compromise): An artifact that indicates a system may have been breached. IOCs include malicious IPs, file hashes (MD5/SHA256), domain names, and URLs observed in attack campaigns.

AbuseIPDB Confidence Score: A community-sourced score (0–100%) representing how likely an IP is malicious, based on number and recency of abuse reports. 90%+ means block immediately.

VirusTotal Engine Detections: Cross-checks an IP/hash against 90+ antivirus and OSINT engines. Even 5–10 detections warrant investigation. 40+ means confirmed malware/botnet involvement.

Attack Categories: The type of malicious activity: DDoS (volumetric attacks), Brute Force (credential attacks), Spam (phishing campaigns), Web App Attack (SQLi/XSS exploitation).

Attributed Campaign / APT Group: Advanced Persistent Threat groups (e.g., Cobalt Mirage, Lazarus Group) are nation-state actors. Attribution helps predict TTPs (Tactics, Techniques, Procedures) from the MITRE ATT&CK framework.

🌐 DNS RECONNAISSANCE EXPLAINED

A Records: Map domain names to IPv4 addresses. Multiple A records reveal load balancing. Use discovered IPs with the IP Geolocation tool.

MX Records: Mail exchange servers. Reveals what email platform the organization uses (Google Workspace, Microsoft 365). Essential for social engineering and phishing campaign targeting.

TXT Records: Contain SPF, DMARC, DKIM policies. No SPF = email spoofing is trivially easy. DMARC = p=reject means spoofed emails are blocked automatically.

NS Records: Nameservers authoritative for the domain. DNS hijacking attacks target the registrar to swap out NS records and gain full control of all DNS.

CAA Records: Certificate Authority Authorization — restricts which CAs can issue SSL certs for this domain. Missing CAA = any CA can issue rogue certs.
